Veloz

Websites & E-commerce

As an intern with The Veloz Group during the summer of 2013, I worked on creating and maintaining a variety of websites for them.

  • The Veloz Group: My first project with Veloz involved some small changes to their home site to familiarize me with their codebase. I learned how to use CodeIgniter, which is a PHP MVC (model-view-controller) framework.
  • Veloz Recruiting: My next project involved working closely with the business interns to revamp the existing Veloz Recruiting site. Goals of the project included increasing usability, efficiently collecting information from potential employers, advertising Veloz interns, and making sure the site displayed well on mobile devices. The existing site met very few of these requirements, so we ended up gutting the old site and rebuilding with a new theme.
  • My Touch Up Paint: The third site I worked on was my first exposure to OpenCart, a PHP MVC framework similar to CodeIgniter but optimized for e-commerce. I worked with one other software intern on sorting out some of the bugs, like a misplaced banner that prevented users from creating an account or logging in, and updating Google Analytics to more effectively track what users were interested in ordering.
  • Fashionable Furnishings: My last project was to work with two other software interns to create a new OpenCart site. A team of business interns was creating a new furniture line, which we dubbed "Fashionable Furnishings" as a working name. They picked an OpenCart theme that our team heavily modified to fit the designers' plans. During this project I learned how to use Inkscape, an open-source design tool like Illustrator; I designed the working logo for Fashionable Furnishings.

Brownian Motion

Websites & Research

Through Fall 2013, I worked with Professor Jose Blanchet under an NSF REU grant. He had worked with a few other students to develop an algorithm that statistically accurately models Brownian motion. I helped develop an application to visualize that algorithm. First I worked with one of the graduate students to understand the MATLAB code and simulations, and then I dove into creating what ended up being a purely JavaScript-based app.

He presented our work under the title "Capturing Infinite Randomness with Computer Simulations," and I will be presenting at a few symposiums at Columbia during Spring 2014.

You can check out the application here and see the source code here.

Orion

Websites & Class Projects

As part of the Open Academy class at Columbia, I have been working with Orion, a cloud-based IDE that's great for writing and testing JavaScript. My team has been working on the Node server for Orion, which is intended for "embedded developers," to quote our mentor. We've been working on a variety of bugs and enhancements, including adding a TTY shell and creating a global installation option for the npm module. You can learn more on our blog, or check out our work on my fork or my teammate's fork.

One of my projects was to create a plugin that clears local storage, so developers writing for Orion Node can see their changes more easily. It's not a default plugin, which means users can navigate to the plugin to install it. That's hosted here. It doesn't look like much, as the meat of the plugin takes place through JavaScript behind the scenes.
